## Ce qu’il faut comprendre tout de suite

- ce projet est un **POC web Vite**,
- il n’a **pas de backend métier dédié**,
- la logique de simulation vit majoritairement côté frontend,
- la progression cloud passe par **Supabase**,
- la conversation prospect est aujourd’hui **heuristique / semi-scriptée**,
- la structure actuelle a été pensée pour pouvoir être transmise puis reprise plus proprement.

## Le point le plus important

Ce projet n’a pas encore vocation à être le moteur final de l’IA ATTRIO.

Il sert aujourd’hui à :

- tester l’expérience,
- valider le parcours de formation,
- structurer les contenus,
- préparer une future intégration plus robuste.
